# Terminal Appearance Settings (themes + fonts)

> Design agreed via brainstorming 2026-06-08. A dedicated "Terminal" settings pane to theme and
> font the xterm.js terminal: curated iTerm2 color schemes + self-hosted monospace fonts, with a
> live preview and app-light/dark sync.

## Problem

The web terminal (`TerminalView`, xterm.js `@xterm/xterm` v6) is constructed with only
`fontFamily: "monospace"` — no color theme (so it always shows xterm's default dark palette
regardless of the app's light/dark mode) and no font/size choice. Users want to theme it (colors +
font + size) from a dedicated settings pane.

## Decisions (from brainstorming)

- **Theme corpus:** a curated ~30 popular schemes (not the full ~390), generated by codegen from the
  mbadolato/iTerm2-Color-Schemes repo's Windows-Terminal JSON. Architecture stays add-a-JSON-and-regen
  so the full set is trivially extensible later.
- **Fonts:** self-hosted woff2 (offline/private), curated set + a System-monospace default. Ligatures
  deferred (needs the ligatures addon + webgl renderer).
- **Live preview:** yes — a non-connected sample xterm in the pane.
- **Default:** "Match app light/dark" (Auto) with a fixed-scheme override.

## 1. Settings pane structure

`SettingsView` becomes **tabbed**: **General** (existing dark-mode switch) + **Terminal** (new pane).
Tab state is local to `SettingsView` (settings sub-views aren't deep-link-critical). `/settings/terminal`
deep-linking is a noted follow-up, not in scope.

## 2. Theme corpus (codegen)

- Vendor the ~30 chosen schemes' Windows-Terminal JSON under `web/scripts/schemes/` (≈1 KB each),
  sourced from mbadolato/iTerm2-Color-Schemes.
- `web/scripts/gen-term-themes.mjs` (npm `gen:themes`): pure, tested `wtToITheme()` maps each WT JSON →
  xterm `ITheme` (background, foreground, cursor, cursorAccent, selectionBackground,
  selectionInactiveBackground, + 16 ANSI: black…white, brightBlack…brightWhite). Writes a **committed**
  `web/src/term/themes.gen.ts` exporting `TERM_THEMES: Record<string, ITheme>` and
  `TERM_THEME_NAMES: string[]`. No network at build/runtime.
- Curated set (~30, adjustable; covers light + dark + famous): Dracula, Solarized Dark, Solarized Light,
  Gruvbox Dark, Gruvbox Light, Nord, One Dark, One Half Light, Tokyo Night, Tokyo Night Storm,
  Catppuccin Mocha, Catppuccin Latte, Monokai Soho, Material, Tomorrow Night, Tomorrow Night Bright,
  Ayu Dark, Ayu Mirage, Ayu Light, Night Owl, Snazzy, Cobalt2, GitHub Dark, Oceanic Next, Seti,
  Argonaut, Espresso (final list confirmed at vendor time against the repo's exact names).

## 3. Fonts (self-hosted)

- woff2 (regular + bold), **unmodified upstream**, under `web/public/fonts/<Font>/`, declared via
  `@font-face` in a CSS file imported once. Families: Fira Code, JetBrains Mono, Source Code Pro,
  Cascadia Code, IBM Plex Mono, Hack, + System monospace (`ui-monospace, SFMono-Regular, Menlo,
  Consolas, monospace`, no file).
- `web/src/term/fonts.ts` registry: `{ id, label, stack }[]`. Browser lazy-fetches only the selected
  family.
- **Font-load correctness:** before xterm measures a freshly-selected font,
  `await document.fonts.load("<px> '<family>'")`, then `fit()` + `term.refresh(0, rows-1)` — else cell
  metrics are wrong.

## 4. Settings store + persistence

`TermSettingsProvider` React context at app root, backed by `localStorage` (key `term-settings`):
```ts
{ follow: boolean;             // "Match app light/dark"
  light: string; dark: string; // scheme names when follow=true
  fixed: string;               // scheme name when follow=false
  fontFamily: string;          // fonts.ts id ("system" default)
  fontSize: number }           // px, default 13
```
Defaults: `follow:true, light:"Solarized Light", dark:"Dracula", fixed:"Dracula", fontFamily:"system",
fontSize:13`. Shared pure/tested helpers used by BOTH the preview and the real terminal:
- `resolveActiveTheme(settings, appDark): ITheme`
- `applyToTerminal(term, settings, appDark)` — sets `term.options.theme/fontFamily/fontSize`.

App light/dark changes: `setTheme()` dispatches an `app-theme-change` event; the context tracks
`appDark` and recomputes the active theme so `follow` mode flips live.

## 5. Terminal pane UI + live preview

Controls: **"Match app light/dark"** toggle → on: *Light scheme* + *Dark scheme* dropdowns; off: single
*Theme* dropdown. Plus *Font family* dropdown and *Font size* (8–24). Below: a **live preview** — a
non-connected `xterm` rendering a fixed sample buffer (prompt line, `ls --color` output, a code snippet,
the 16 ANSI swatches), re-applying settings on every change via `applyToTerminal`.

## 6. TerminalView integration

`TerminalView` consumes the context: builds its terminal with the resolved theme/font/size, and a
**separate effect keyed on `[settings, appDark]`** mutates `term.options` live (loads the font, `fit()`s,
re-sends the resize) **without recreating the terminal** (preserves scrollback). The spawnId-keyed socket
effect is untouched. Applies to all current/future terminals (sp-npxq tab bar).

## 7. Licensing & attribution

- **Fonts** (copyrightable): all chosen families are SIL OFL 1.1 (Fira Code, JetBrains Mono, Source Code
  Pro, Cascadia Code, IBM Plex Mono) or the permissive Hack Open Font License — embedding/redistribution
  is permitted. Obligations: (a) **bundle each font's license + copyright** in `web/public/fonts/<Font>/`;
  (b) **ship unmodified upstream binaries — do NOT subset/modify** (avoids the OFL Reserved-Font-Name
  rename obligation and gives full glyph coverage); (c) don't sell the fonts standalone (we don't).
- **Color schemes** (data — low risk): hex palettes are essentially data; keep scheme **names** and add
  attribution rather than relying on "it's just data."
- **`web/THIRD_PARTY_NOTICES.md`**: credits the font families + their licenses, and the color schemes
  (mbadolato repo + per-scheme authors where listed). Each font/scheme vendoring task's definition of
  done includes "license/notice present; upstream binary unmodified."

## 8. Testing

- **vitest:** `wtToITheme` mapper; `themes.gen.ts` sanity (expected names present, valid hex per ITheme
  field); `resolveActiveTheme` (follow vs fixed × light/dark); store load/save/defaults; Terminal pane
  renders controls + updates the store; preview + `TerminalView` apply settings to a mocked `Terminal`
  (assert `options.theme/fontFamily/fontSize` + refit). xterm mocked as in `TerminalView.test.tsx`.
- **e2e:** light/optional (canvas is hard to assert) — verify the pane's selects update + persist; skip
  pixel assertions.

## Out of scope / follow-ups

- Ligatures (ligatures addon + webgl renderer).
- `/settings/terminal` deep link.
- Per-scheme color swatches in the dropdown; searchable picker if the set grows large.
- The full ~390 corpus (extensible: vendor JSON + rerun `gen:themes`).
