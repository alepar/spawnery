# sp-9xr.15 — Anthropic↔OpenAI converter (Claude Code via the OpenRouter sidecar) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Let Claude Code (which speaks the **Anthropic Messages API**) run against OpenRouter (OpenAI-only) by adding a `/v1/messages` translation endpoint to the sidecar: Anthropic Messages request → OpenAI Chat Completions → OpenRouter, and the response back (incl. **streaming SSE + tool_use**). Then wire the dispatcher's `claude-tui` case to point Claude Code at it. Verify a real Claude-Code turn (including a tool call) completes through it.

**Architecture:** A new Go converter in `internal/sidecar` exposing `http.Handler` for `POST /v1/messages`. `cmd/sidecar/main.go` becomes a mux: `/v1/messages` → converter; everything else → the existing transparent `NewHandler` (OpenAI passthrough). The converter forwards to `<upstream>/v1/chat/completions` with the injected key. The dispatcher's `claude-tui` case sets `ANTHROPIC_BASE_URL`=sidecar + a dummy key + the model + headless flags. No Python (consistent with the nori choice) — pure Go.

**Tech Stack:** Go (`internal/sidecar`), Docker, real OpenRouter key. **Run Go:** `export PATH="/var/home/alepar/sdk/go1.26.0/bin:$PATH"`.

### Context (verified)
- `internal/sidecar/proxy.go` `NewHandler(upstream, key)` = transparent reverse proxy to OpenRouter, injects `Authorization: Bearer <key>`, deletes `X-Api-Key`. `cmd/sidecar/main.go` serves it on `SIDECAR_ADDR` (127.0.0.1:8080) with `SIDECAR_UPSTREAM=https://openrouter.ai/api`.
- OpenRouter: OpenAI-only, `POST {upstream}/v1/chat/completions`, Bearer auth (confirmed).
- Claude Code (from `.13`/`.15` docs research): speaks Anthropic Messages API to `ANTHROPIC_BASE_URL` (it POSTs `{ANTHROPIC_BASE_URL}/v1/messages`), always **streams**, uses **tools** heavily. Config: `ANTHROPIC_API_KEY`=dummy (sidecar injects real), `ANTHROPIC_CUSTOM_MODEL_OPTION=<openrouter-model-id>` (+`_NAME`), `ENABLE_TOOL_SEARCH=true`, `DISABLE_AUTOUPDATER=1`, `DISABLE_TELEMETRY=1`, `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1`. Non-interactive: `claude -p "<prompt>"`.
- The agent container has `OPENAI_BASE_URL=http://<sidecar>:8080/v1` + `SPAWN_MODEL`; sidecar addr is reachable in-pod.
- claude-tui is currently a dispatcher stub (`exit 1`) in `deploy/agent/entrypoint.sh`. agentcaps has `claude-tui` (ModeTmux).

### Scope of the translation (the subset Claude Code needs)
**Request (Anthropic→OpenAI):** `model`; `system` (string OR array of text blocks) → a leading `{"role":"system"}` message; `messages[]` where content is a string OR blocks: `text`→text, `tool_use`{id,name,input}→ assistant `tool_calls[]`{id,type:function,function:{name,arguments:JSON.stringify(input)}}, `tool_result`{tool_use_id,content}→ `{"role":"tool","tool_call_id":...,"content":...}`, `image`→ OpenAI image_url (best-effort; ok to drop if absent); `tools[]`{name,description,input_schema}→`[{type:function,function:{name,description,parameters:input_schema}}]`; `tool_choice` map (auto/any/tool→auto/required/{specific}); `max_tokens`→`max_tokens`; `temperature`/`top_p`; `stop_sequences`→`stop`; `stream`. IGNORE Anthropic extras (`cache_control`, `thinking`, `metadata`).
**Non-streaming response (OpenAI→Anthropic):** `{id, type:"message", role:"assistant", model, content:[ {type:text,text} for message.content, {type:tool_use,id,name,input:JSON.parse(arguments)} for each tool_call ], stop_reason (stop→end_turn, length→max_tokens, tool_calls→tool_use), usage:{input_tokens, output_tokens}}`.
**Streaming response (OpenAI SSE → Anthropic SSE):** emit, in order: `message_start` (with the message skeleton + role:assistant) → for the first text delta `content_block_start`{index:0,type:text} then `content_block_delta`{index:0,delta:{type:text_delta,text}} per chunk → for tool_calls: `content_block_start`{index:N,type:tool_use,id,name} then `content_block_delta`{index:N,delta:{type:input_json_delta,partial_json}} accumulating `function.arguments` fragments → `content_block_stop` per closed block → `message_delta`{delta:{stop_reason,...},usage} → `message_stop`. Send `event: <type>\ndata: <json>\n\n` framing. Optionally periodic `event: ping`. Handle `data: [DONE]`.
