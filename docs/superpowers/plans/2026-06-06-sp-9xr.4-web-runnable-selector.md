# sp-9xr.4 — Web two-dropdown agent selector (image → runnable) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let a user pick the agent **image** and **runnable** when spawning an app. The CP returns derived runnables (server-side `agentcaps` expansion); the Detail view shows two `<select>` dropdowns when runnable-bearing images exist; the choice flows into `createSpawn(appId, model, image, runnableId)`. When no runnable-bearing images are registered, the flow is unchanged (legacy spawn).

**Architecture:** Extend `AgentImageInfo` with `repeated RunnableInfo runnables` and expand binaries→runnables in the `ListAgentImages` handler via `agentcaps`. Web: `listAgentImages()` returns runnables; `createSpawn` gains optional `image`/`runnableId`; `Detail.tsx` renders the cascading selects and passes the selection up the `onSpawn` chain to `App.tsx`.

**Tech Stack:** Go (Connect, `agentcaps`, `buf`), web TypeScript (React, vite, vitest, testing-library).

**Toolchain:** Go: `export PATH="/var/home/alepar/sdk/go1.26.0/bin:$HOME/go/bin:$PATH"` (buf+plugins for regen). Web tests: `cd web && npx vitest run` (or `./node_modules/.bin/vitest run`).

**Spec:** `docs/superpowers/specs/2026-06-06-tmux-terminal-mode-design.md` (§"Selection & lifecycle" — "image dropdown → runnable dropdown (friendly labels); auto-select singletons").

In `web/src/api/spawnlet.ts`, replace the `AgentImageView`/`listAgentImages` block and extend `createSpawn`:

```ts
export interface RunnableView {
  id: string;
  label: string;
  mode: string;
}
export interface AgentImageView {
  image: string;
  runnables: RunnableView[];
}
export async function listAgentImages(): Promise<AgentImageView[]> {
  const r = await unary<{ images?: Array<{ image?: string; runnables?: Array<{ id?: string; label?: string; mode?: string }> }> }>(
    "ListAgentImages", {},
  );
  return (r.images ?? []).map((i) => ({
    image: i.image ?? "",
    runnables: (i.runnables ?? []).map((x) => ({ id: x.id ?? "", label: x.label ?? "", mode: x.mode ?? "" })),
  }));
}
```

```ts
export async function createSpawn(appId: string, model: string, image = "", runnableId = ""): Promise<string> {
  const r = await unary<{ spawnId: string }>("CreateSpawn", { appId, model, image, runnableId });
  return r.spawnId;
}
```
