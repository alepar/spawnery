# sp-9xr.18 — tmux/terminal-mode polish (4 fixes) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix four bugs found running terminal-mode (tmux) spawns in the web UI + `spawnctl attach`:
1. **opencode-tui bypasses the sidecar** (uses opencode's built-in provider/zen; autogenerated session name).
2. **Web status dot wrong for tmux mode** (stays hidden — `TerminalView`'s socket never feeds the dot).
3. **Claude Code "Detected a custom API key in your environment" warning** at startup.
4. **Broken glyphs** — tmux shows VT100 ACS line-drawing as `q`/`x`; claude ASCII-substitutes `●`→`_` in the web. Root cause: no UTF-8 locale in the image.

**Tech Stack:** Docker image (`deploy/agent/`), Go (`internal/spawnlet/terminal.go`), web TS (`web/src/`). Real Docker + OpenRouter for live verification. **Run Go:** `export PATH="/var/home/alepar/sdk/go1.26.0/bin:$PATH"`. Image build is slow (~minutes); use a long timeout. Real `OPENROUTER_API_KEY` is in the repo-root `.env` — handle as a secret (never print/commit); source it into the e2e test env as prior implementers did.

### Context (verified)
- **Dispatcher** `deploy/agent/entrypoint.sh`: the default `""|opencode-served` body (after the `case`) generates `/etc/opencode/opencode.json` (custom `spawnery` provider → sidecar `SIDECAR_BASE`), sets `OPENCODE_CONFIG`, and exports `SPAWN_MODEL=spawnery/<MODEL_ID>` then runs `opencode serve` + `acpadapter`. The `opencode-tui)` case is just `export TERM; exec spawn-tmux opencode` — it does NONE of that config-gen, so the TUI uses opencode's built-in provider. The `claude-tui)` case `export ANTHROPIC_API_KEY=sk-unused-sidecar-injects-real-key` (+ `ANTHROPIC_BASE_URL`, model options) then `exec spawn-tmux claude`.
- **`deploy/agent/spawn-tmux`**: `tmux new-session -d -s "$SESSION" -- "$@"` (no `-u`). **`deploy/agent/Dockerfile`**: `debian:stable`, NO `LANG`/locale set (container locale = POSIX); `C.utf8` IS available in glibc (`locale -a` shows it) — no `locales` package needed.
- **`internal/spawnlet/terminal.go` `ExecPrefixFor`**: docker lane `["docker","exec","-it","-e","TERM=xterm-256color"]`; runsc lane `["crictl","exec","-it"]`. Forwards `TERM` but not `LANG`. (docker exec inherits image ENV; crictl may not — forward `LANG` explicitly on both to be safe.)
- **Web status dot**: `web/src/shell/ConnStatus.tsx` renders the dot from a `ConnState | null` `conn` prop (null → hidden). `conn` comes from `useConnStatus()` in `App.tsx` (`connected()/reconnecting()/waiting()/connecting()/errored()/reset()`), driven by the ChatView ACP socket in `openSession()`. For tmux: `App.tsx:152` and `:233` gate `openSession` OFF (`mode !== "tmux"`), and `TerminalView` (`web/src/views/TerminalView.tsx`) opens its OWN `/ws/session` socket with an empty `onDown` and never reports up → `conn` stays `null` → dot hidden. `AppShell.tsx:51` renders `<TerminalView spawnId={activeId} />` for `activeMode === "tmux"`.
- The live e2e guards: `internal/cp/acp_e2e_test.go` / `tmux_e2e_test.go` — `TestCPClaudeTuiEndToEnd` (real claude tool round-trip), `TestCPTmuxEndToEnd`/`TestCPGooseTuiEndToEnd`. These must stay green.

## Task 1: UTF-8 locale (fix #4 — `q`/`x` in tmux, `●`→`_` in web)

**Files:** `deploy/agent/Dockerfile`, `deploy/agent/spawn-tmux`, `internal/spawnlet/terminal.go`

- [ ] **Step 1: Set a UTF-8 locale in the image.** In `deploy/agent/Dockerfile` final stage, add (near the top of the final stage, before the agent installs is fine):
  ```dockerfile
  ENV LANG=C.UTF-8 LC_ALL=C.UTF-8
  ```
  (`C.UTF-8`/`C.utf8` is built into glibc — verified `locale -a` lists `C.utf8`; no `locales` package needed.)
- [ ] **Step 2: Force tmux UTF-8.** In `deploy/agent/spawn-tmux`, add `-u` to the new-session: `tmux -u new-session -d -s "$SESSION" -- "$@"` (belt-and-suspenders alongside the locale).
- [ ] **Step 3: Forward LANG on terminal attach.** In `internal/spawnlet/terminal.go` `ExecPrefixFor`, add `-e LANG=C.UTF-8 -e LC_ALL=C.UTF-8` to the docker lane, and for the crictl lane add the same `-e` flags (crictl exec supports `-e`? VERIFY; if crictl lacks `-e`, the image ENV covers the runsc lane already — note it). Update any `ExecPrefixFor` test expecting the exact argv (`internal/spawnlet/*_test.go`).
- [ ] **Step 4: Build + verify glyphs live.**
  ```bash
  docker build -t spawnery/agent:dev -f deploy/agent/Dockerfile .
  docker run --rm --entrypoint sh spawnery/agent:dev -c 'echo LANG=$LANG; locale | head -2'
  # claude-tui in tmux must render UTF-8 box/●, NOT q/x/_:
  cid=$(docker run -d -e SPAWN_MODEL=openai/gpt-4o-mini -e OPENAI_BASE_URL=http://127.0.0.1:8080/v1 --entrypoint /usr/bin/tini spawnery/agent:dev -- /entrypoint.sh claude-tui); sleep 5
  docker exec "$cid" tmux capture-pane -p -t spawn | head -8
  docker rm -f "$cid"
  ```
  Expected: `LANG=C.UTF-8`; capture-pane shows `─`/`│`/`●` (UTF-8), NOT the `qqq`/`xxx` ACS pattern. Paste output. (claude may show its own startup screen; the point is the box chars render as UTF-8.)

- [ ] **Step 5: Commit.** `git add deploy/agent/Dockerfile deploy/agent/spawn-tmux internal/spawnlet/ && git commit -m "fix(agent): UTF-8 locale (C.UTF-8) so tmux+claude render box/glyphs not q/x/_ (sp-9xr.18)"`
