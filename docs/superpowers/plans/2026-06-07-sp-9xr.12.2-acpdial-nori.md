# sp-9xr.12.2 — acpdial shim + nori terminal attach Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** `spawnctl attach` to an **acp-mode** spawn launches **nori** (the Rust ACP TUI) inside the container, joining the SAME shared `goose acp` conversation the web ChatView is on — so one user can drive the agent from the terminal on one device and the web on another. nori can't dial an external ACP server, but it spawns its agent as a child over stdio; so we configure nori's custom agent to launch a tiny **`acpdial`** stdio↔TCP shim that connects to `acpmux` on `127.0.0.1:7000` (the in-container multiplexer from sp-9xr.12.1). nori thinks it spawned an agent; really it's now a second ACP client of `acpmux` → shared session. No nori fork.

**Architecture:**
- **`acpdial`** (new tiny Go binary, the inverse of `acpexec` from sp-9xr.16): pipe `os.Stdin`/`os.Stdout` ↔ a TCP dial to `127.0.0.1:7000` (configurable via arg/env). ~30 lines.
- **nori custom agent**: a baked `config.toml` (and/or `-c` overrides) defining a "spawnery" agent whose local distribution command is `acpdial` (stdio ACP). `terminalInnerCmd` runs `nori` with that agent + the non-interactive flags.
- **Wiring**: `internal/spawnlet/terminal.go` `terminalInnerCmd(sp)` gains an `acp` branch → the nori launch argv (today acp-mode wrongly falls to `spawn-tui`). The node's existing terminal path (`StartTerminal` → mosh-server → `docker exec -it <c> <inner>`) carries nori's TUI over the mosh PTY. **spawnctl needs no change** — the node picks the command by `sp.Mode`.

**Tech Stack:** Go (`deploy/agent/acpdial` + `internal/spawnlet/terminal.go`), nori config (TOML baked in the image), Docker, real Docker + OpenRouter. **Run Go:** `export PATH="/var/home/alepar/sdk/go1.26.0/bin:$PATH"`.
