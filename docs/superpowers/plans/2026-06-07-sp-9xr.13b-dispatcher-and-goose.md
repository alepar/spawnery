# sp-9xr.13b — Dispatcher entrypoint + wire opencode & goose-tui Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Realize the "image owns launch, CP/node pass the runnable name" contract: `entrypoint.sh` becomes a dispatcher keyed by the runnable id; the node manager passes the runnable id as the container command (tmux-wrapping moves OUT of the manager INTO the dispatcher). Wire the cases that work today — `opencode-served` (default ACP), `opencode-tui` (tmux), and `goose-tui` (tmux, goose pointed at the OpenAI sidecar) — and stub the rest (`goose-acp`→sp-9xr.16, `claude-tui`→sp-9xr.15, `toad-*`→sp-9xr.12) with a clear error.

**Architecture:** `deploy/agent/entrypoint.sh` dispatches on `$1` (runnable id): empty/`opencode-served` → the existing opencode serve+adapter path; `opencode-tui` → `spawn-tmux opencode`; `goose-tui` → set goose's OpenAI-sidecar env then `spawn-tmux goose`; unwired → exit 1 with a pointer. The manager (`CreateWithSelection`) sets `AgentSpec.Cmd = [sel.RunnableID]` for any selection (the dispatcher resolves the actual launch), `nil` for none. `agentcaps.Launch` becomes reference-only (the image owns launch).

**Tech Stack:** Docker image (deploy/agent), Go (spawnlet manager), real Docker + OpenRouter for the e2e. **Run Go:** `export PATH="/var/home/alepar/sdk/go1.26.0/bin:$PATH"`.

### Context (verified)
- The agent container already gets `OPENAI_BASE_URL=http://<sidecar>:8080/v1` and `SPAWN_MODEL=<id>` from the manager (`manager.go` AgentSpec.Env). opencode's `entrypoint.sh` builds a `spawnery` OpenAI-compatible provider at `SIDECAR_BASE` (127.0.0.1:8080/v1) with a dummy key.

## Task 1: Dispatcher entrypoint + manager passes the runnable id

**Files:** `deploy/agent/entrypoint.sh`, `internal/spawnlet/manager.go`, `internal/spawnlet/manager_selection_test.go`

- [ ] **Step 1: Rewrite entrypoint.sh as a dispatcher**

Replace the `.5` argv passthrough block with a `case` on `$1`. Keep the existing opencode serve+adapter logic as the default/`opencode-served` body:

```sh
set -e

# Dispatcher: the node passes the runnable id as $1; the image owns how each runnable launches.
# Empty (legacy) or opencode-served -> the default opencode serve + adapter path below.
case "${1:-}" in
  ""|opencode-served)
    : # fall through to the default opencode path below
    ;;
  opencode-tui)
    export TERM="${TERM:-xterm-256color}"
    exec spawn-tmux opencode
    ;;
  goose-tui)
    export TERM="${TERM:-xterm-256color}"
    # goose -> the OpenAI-compatible sidecar (same endpoint opencode uses). VERIFY the exact goose
    # env var names against `goose` in the image (goose's openai provider host/key/model) — the e2e
    # must show goose actually reaching the sidecar.
    export GOOSE_PROVIDER="${GOOSE_PROVIDER:-openai}"
    export GOOSE_MODEL="${SPAWN_MODEL:-openai/gpt-4o-mini}"
    export OPENAI_API_KEY="${OPENAI_API_KEY:-sk-unused-sidecar-injects-real-key}"
    # OPENAI_BASE_URL is already set by the node to the sidecar; map to whatever goose expects
    # (OPENAI_HOST / OPENAI_BASE_PATH) — confirm in-container.
    exec spawn-tmux goose
    ;;
  goose-acp)
    echo "goose-acp not wired yet (needs stdio-ACP->TCP bridge; see sp-9xr.16)" >&2; exit 1 ;;
  claude-tui)
    echo "claude-tui not wired yet (needs Anthropic<->OpenAI converter; see sp-9xr.15)" >&2; exit 1 ;;
  toad-*|toad)
    echo "toad not wired yet (ACP-TUI client; see sp-9xr.12)" >&2; exit 1 ;;
  *)
    echo "unknown runnable: $1" >&2; exit 1 ;;
esac

# --- default opencode serve + adapter (runnable "" / opencode-served) ---
SIDECAR_BASE="${SIDECAR_BASE:-http://127.0.0.1:8080/v1}"
# ... (KEEP the existing opencode.json generation + opencode serve + exec acpadapter exactly as-is) ...
```

(Preserve the existing opencode config-gen + `opencode serve` + `exec acpadapter` lines verbatim as the default body.)
