> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `spawnctl attach` to a **tmux**-mode spawn drops you into the agent's tmux session (`tmux attach -t spawn`) over the existing mosh terminal lane, instead of the hardcoded opencode-attach (`spawn-tui`). Non-tmux spawns keep today's behavior.

**Architecture:** Carry the spawn's `Mode` on the node-side `spawnlet.Spawn` (set at create). The terminal launcher picks the in-container inner command by mode: tmux → `tmux attach -t spawn`; else → the existing `spawn-tui`. The mosh wiring is unchanged (same lane that already works for opencode-attach).

**Tech Stack:** Go (`internal/spawnlet`), mosh + Docker (verifiable here). **Run Go:** `export PATH="/var/home/alepar/sdk/go1.26.0/bin:$PATH"`.

### Context (verified)
- `internal/spawnlet/store.go` `Spawn` struct has no `Mode`.
- `internal/spawnlet/manager.go` `CreateWithSelection` builds the `&Spawn{...}` near the end (with `Status: "ready"`); `sel.Mode` is in scope.
- `internal/spawnlet/terminal.go`: `attachCommand()` returns `["spawn-tui"]`; `(m *Manager) StartTerminal(ctx, spawnID, cmd)` looks up `sp` then calls the free `StartTerminal(...)` with `cmd` (nil → `attachCommand()`). The tmux session created by `.5`'s `spawn-tmux` is named `spawn`.
- spawnctl: `cmd/spawnctl/terminalcmd.go` POSTs `/terminal?spawn=<id>` with optional `{"cmd":[...]}`; nil cmd = default attach. No spawnctl change needed — the node picks the default per mode.

- [ ] **Step 4: Add `terminalInnerCmd` + use it in StartTerminal**

In `internal/spawnlet/terminal.go`, add:
```go
// terminalInnerCmd is the in-container command a default terminal attach runs for a spawn: tmux-mode
// spawns attach to the in-container tmux session (created by spawn-tmux); all others get the opencode
// TUI launcher.
func terminalInnerCmd(sp *Spawn) []string {
	if sp.Mode == "tmux" {
		return []string{"tmux", "attach", "-t", "spawn"}
	}
	return attachCommand()
}
```
Then in `(m *Manager) StartTerminal`, when the caller passed no explicit `cmd`, default to the mode-aware command. Change the manager method so that an empty `cmd` becomes `terminalInnerCmd(sp)` before calling the free `StartTerminal`:
```go
	if len(cmd) == 0 {
		cmd = terminalInnerCmd(&sp)   // sp is the looked-up *Spawn (adjust deref to match the local)
	}
	return StartTerminal(ctx, sp.AgentID, cmd, TerminalConfig{...})
```
(The free `StartTerminal`'s own `len(inner)==0 → attachCommand()` fallback stays as a safety net for direct callers, but the manager now passes the mode-aware cmd explicitly.)
